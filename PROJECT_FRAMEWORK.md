# PROJECT_FRAMEWORK.md

## 当前项目主框架

仓库当前只保留一条有效前端主线：

1. `frontend/`
   - Vue 3 + TypeScript + Vite
   - 当前唯一有效的管理端主线
   - 后续后台壳层、页面、路由、状态与样式演进都以此目录为唯一落点

## 当前前端实施约束

- 所有管理端页面、壳层、路由、状态与样式调整默认在 `frontend/` 内进行。
- 不再新增第二条管理端工程，不再并行维护第二套后台前端主线。
- 优先延续现有信息架构、路由职责和页面边界，不做无约束的大面积推翻。
- 当前阶段默认继续使用 mock / adapter 骨架，不接真实 API、真实登录和真实权限。
- 页面演进顺序保持为“稳定壳层优先、内容页逐步替换、公共能力再抽象”。

## 实施顺序

1. 先阅读 `frontend/` 现状，确认信息架构、页面边界和公共壳层规律。
2. 在现有主线内逐步收口主题、组件和页面骨架，不额外拆分第二工程。
3. 优先替换内容区与页面工作面，再评估是否需要抽取公共容器、共享状态或数据边界。
4. 在壳层稳定后再逐步接入真实 API 和上下文同步。

## 当前非目标

- 不在本期接入真实后端 API。
- 不在本期做真实登录。
- 不在本期做真实权限校验。
- 不在本期开启新的独立前端重写目录或第二后台工程。
