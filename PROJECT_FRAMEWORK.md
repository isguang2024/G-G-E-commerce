# 项目主框架与执行基线

> 现状基线：2026-03-27。本文只保留当前长期有效的主架构、边界与执行清单，不再保留阶段性方案描述。

## 1. 文档定位

- `PROJECT_FRAMEWORK.md` 只定义全局稳定约束，不承载模块级实现细节。
- 模块实现、专题规则、迁移说明放在 `docs/`。
- 若文档与代码冲突，以代码真实行为为准，然后回补文档。

适用场景：
- 新增业务模块
- 修改权限、菜单、页面、API、迁移主链
- 涉及前后端联动的结构性改造

不适用场景：
- 局部样式微调
- 单个函数重命名
- 一次性排障记录

## 2. 当前架构总览

### 2.1 平台形态

本项目是“平台 + 团队”双上下文电商基础平台，不是静态页面集合。

当前稳定底座包括：
- 认证与身份：`users`、`roles`、`tenants`、`tenant_members`
- 功能包：`feature_packages`、`feature_package_bundles`
- 权限键：`permission_keys`
- 菜单：`menus`、`menu_manage_groups`
- 页面：`ui_pages`
- API 元数据：`api_endpoints`、`api_endpoint_permission_bindings`
- 运行时快照：
  - `platform_user_access_snapshots`
  - `platform_role_access_snapshots`
  - `team_access_snapshots`
  - `team_role_access_snapshots`

### 2.2 当前运行时主链

运行时主链已经收口为：

1. API 请求先经过 API 状态缓存，已停用接口直接拦截。
2. 鉴权按是否存在 `X-Tenant-ID` 区分 `platform` / `team`。
3. 平台侧优先读取平台用户/平台角色快照。
4. 团队侧优先读取团队边界快照与团队角色快照。
5. 前端运行时路由由“静态路由 + 菜单路由 + 运行时页面路由”组成。

### 2.3 当前稳定主数据语义

- 功能包是业务开通单元。
- 权限键是动作/API 的原子能力标识。
- 菜单只负责导航入口，不再承载“内页类型”主链语义。
- 页面是访问单元，负责路径、挂载、面包屑和访问模式。
- API 元数据是接口边界、权限归属和运行时状态控制入口。

## 3. 稳定边界

### 3.1 平台与团队

- `platform` 与 `team` 必须显式区分。
- 仅团队上下文传 `X-Tenant-ID`。
- 不能把平台权限、团队边界、团队角色混成一套隐式逻辑。

### 3.2 菜单、页面、权限、功能包

- 菜单控制导航入口。
- 页面控制访问单元与挂载结构。
- 权限键控制动作/API。
- 功能包控制模块开通范围。

禁止：
- 前端绕过后端语义自行补权限。
- 菜单直接等于页面。
- 页面直接等于权限。
- 在业务模块复制平行权限体系。

### 3.3 兼容层

- 兼容字段和兼容表允许短期保留，但不能继续扩张。
- 历史兼容逻辑优先下沉到迁移和收敛层，运行时只保留最小必要兼容。

## 4. 当前模块组织约束

- `backend/internal/modules/system/**` 承载平台基础能力。
- 业务模块应复用 `system` 已有的认证、权限、菜单、页面、API 注册与快照刷新能力。
- 前端管理页默认遵循：
  - [FRONTEND_GUIDELINE.md](/C:/Users/Administrator/Documents/GitHub/G-G-E-commerce/FRONTEND_GUIDELINE.md)
  - 页面/菜单/权限链路消费后端正式接口，不自行拼接规则

## 5. 执行清单

### 5.1 输入检查

提交前必须明确：
- 影响上下文：`platform` / `team` / `platform,team`
- 影响对象：菜单 / 页面 / 功能包 / 权限键 / API 元数据 / 快照刷新
- 是否涉及：表结构变更 / 数据迁移 / 兼容逻辑下线

### 5.2 设计检查

- 是否复用既有认证、权限、租户、页面、API 注册能力
- 是否避免引入新的平行模型
- 是否明确前端、后端、迁移、文档的影响范围
- 是否把临时补丁收敛为可维护机制

### 5.3 实施检查

- 接口语义是否与前端消费一致
- 迁移是否幂等、可重复执行
- 快照刷新是否覆盖受影响主体
- 文档是否同步到根文档或专题文档

## 6. 完成标准与验证矩阵

满足以下条件才算完成：
- 核心链路至少完成最小冒烟
- 已明确写出已验证项与未验证项
- 文档入口已同步更新

最低验证矩阵：
- 后端结构、权限、迁移改动：`go test ./...`
- 前端页面、交互、路由改动：`pnpm exec vue-tsc --noEmit`
- 同时涉及前后端：两者都执行

未验证项固定格式：
- `未验证：<命令/场景>，原因：<未执行原因>`
