# 下次方向

- 新开 Codex 会话或重启应用后，再确认当前仓库能正常发现 `superpowers` 技能。
- 如果后续希望所有仓库都能使用这套技能，再考虑同步到用户级技能目录，而不是只保留项目级接入。
- 如果实际使用中觉得流程过重，优先裁剪为 `systematic-debugging`、`verification-before-completion`、`requesting-code-review`、`dispatching-parallel-agents` 等高收益技能组合。
- 当前 `frontend/` 默认没有 `test` 脚本，使用 `test-driven-development` 时需要结合仓库实际能力，不要机械套用上游默认流程。
