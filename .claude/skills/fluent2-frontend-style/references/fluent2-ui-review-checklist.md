# Fluent 2 界面评审清单

## 焦点与层级

- 主任务是否能在首屏一眼识别
- 主操作是否比次级操作更明确
- 是否存在多个区域同时争抢注意力
- 次级信息是否被正确下沉

## 结构与布局

- 应用壳是否稳定
- 页面头部是否只承载高价值信息
- 主工作区是否足够集中
- 次级面板是否承担辅助而非主任务

## 文案与状态

- 标题是否像任务名而不是广告语
- 状态是否被清晰表达
- 错误、警告、成功反馈是否在正确位置
- 危险操作是否写清后果

## 视觉与品牌

- 颜色是否克制且有明确用途
- 边框、阴影、背景是否服务于层级而不是装饰
- 是否具有稳定、专业、微软式产品气质
- 是否存在营销页式夸张视觉混入工作台

## 交互与可访问性

- 键盘焦点是否连续可见
- 交互反馈是否及时且不过度
- 动效是否帮助理解层级变化
- 响应式下是否仍保留主要工作流
