# Fluent 2 原则摘要

## 核心原则

- `Natural on every platform`
  先顺应当前平台的阅读、触控、鼠标和键盘习惯，再谈品牌风格。

- `Built for focus`
  先把主任务、主操作和主信息排清楚，再决定装饰和视觉强化。

- `One for all, all for one`
  局部页面要服从整体系统，一套产品不要出现多种互相竞争的界面语言。

- `Unmistakably Microsoft`
  气质要可靠、克制、专业、清晰，强调效率和信息表达，而不是炫耀式视觉。

## 设计语气

- 干净，但不空洞。
- 信息密，但不拥堵。
- 专业，但不冷漠。
- 有品牌感，但不靠装饰堆砌。

## 对后台页面的翻译

- 页面先表达任务和状态，再表达氛围。
- 让高频动作贴近数据和对象本身。
- 用分割、间距和字号建立层次，避免全靠卡片包裹。
- 在复杂页面里保留稳定的导航、属性区和反馈区。

## 相关系统页面

- `Handoffs`：用于跨页面或跨应用的上下文交接
- `Wait UX`：用于等待、加载、生成和同步时的反馈
- `Motion`：用于层级变化、过渡和状态转场
- `Accessibility`：用于文本缩放、对比度、扫读和焦点秩序
- `Design tokens`：用于颜色、间距、圆角、阴影和字号收口

## 常见误区

- 误把 Fluent 2 理解成“全白背景 + 轻边框”。
- 误把 Microsoft 气质理解成“桌面软件截图感”。
- 误把简洁理解成“删掉说明、删掉反馈、删掉层级”。
