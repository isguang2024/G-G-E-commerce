# 后台与工作台页面配方

## 登录页

- 用品牌区、欢迎语、表单区三段式组织。
- 保持表单区简洁，只保留登录必需字段和单条辅助说明。
- 登录失败在字段附近或表单顶部给出明确错误，不要只弹全局提示。

## 应用壳

- 顶部栏放全局动作、搜索、环境信息和用户菜单。
- 左侧导航承载模块切换和层级结构。
- 内容区固定留给页面标题、摘要、工具栏和主工作区。
- 次级面板用于属性检查、详情或上下文辅助，不抢主工作区焦点。

## 列表页

- 顺序固定为：标题区、筛选区、工具栏、表格、分页或详情。
- 高价值状态用 `Badge`、`Tag`、`MessageBar` 表达，不靠颜色块堆砌。
- 行级操作保持少而明确，低频操作放进菜单。

## 详情或编辑页

- 用 section 拆分信息，不做一整面无层次长表单。
- 主操作固定放底部或顶部工具栏一处，不要上下重复。
- 危险操作单独分区，并给出后果说明。

## 设置页

- 将设置项按主题、权限、通知、集成等分组。
- 每组只保留一个明确目标，避免同屏出现过多无关开关。
- 对影响系统行为的开关，补充说明文字与即时反馈。
