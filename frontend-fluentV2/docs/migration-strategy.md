# 迁移策略

## 核心原则

- 先搭基础壳，再逐页迁移
- 先建立通用壳层能力，再接 API
- 先保留信息架构，再替换内容实现
- 不复刻旧视觉，不翻译旧 Vue 代码

## 建议顺序

1. 系统治理主链：菜单、页面、API 端点、角色、用户
2. 工作台类页面：首页、收件中心、消息工作区
3. 团队协作链路：团队总览、成员、角色边界
4. 真实数据接入：菜单空间、当前上下文、导航权限、接口 adapter

## 接口替换原则

- 保持 Query key 不变
- 保持 `PageContainer` 与 `AppShell` 不变
- 只替换 mock 查询函数或 `api/adapter`
- 不让页面重新承担导航、空间或上下文的全局职责
