# FRONTEND_GUIDELINE.md

适用范围：`frontend/`（Vue 3 + TS + Vite + Element Plus + Pinia）。

## 数据与接口

- **已接真实接口**，不再使用 mock。
- API 客户端由后端 OpenAPI spec 生成，统一放在 `frontend/src/api/`，**禁止手写 axios 类型与请求层**。
- 后端契约变更 → 重新生成 client → 前端同步更新；不允许前端临时改字段绕过 spec。
- 路由守卫与权限判断消费后端 `/permissions/explain` 真实结果，不在前端手写权限规则。
- 前端不感知 `tenant_id`，由后端中间件解析。

## 视觉方向

- 克制、清晰、低噪。
- 先用留白、层级和排版组织信息，不堆卡片皮肤。
- 后台场景优先稳定感与可读性，不做营销化堆叠。

## 壳层规范

- 顶部栏：品牌、当前区域、空间切换、主题入口、用户区、预留全局操作。
- 左侧导航：模块切换与层级定位，不堆业务操作。
- 内容区：统一面包屑、标题、副标题、页面容器。
- 未实现页面进入统一占位页，禁止白屏。

## 页面组织

- 标题区、操作区、内容区分层明确。
- 接口调用、样式、状态、布局不揉在一个文件。
- 列表、设置、三栏工作区尽量复用统一容器。
- 新页面延续现有信息架构，不用"新目录 + 新实现"绕开主线。

## 组件与样式

- `frontend/` 内保持单一 UI 体系，不混用多套设计语言。
- 配色、圆角、间距、阴影统一通过 token 或主题层管理。
- Tailwind 与 Element Plus 共存时，公共布局抽到统一层，不让页面散落魔法类名。
- 新增基础组件优先沉淀为可复用模块。

## 动效

- 最小必要原则，主要服务导航切换、抽屉/弹层、状态反馈。
- 自定义过渡统一时长与缓动，不在不同页面各写一套节奏。

## 状态管理

- Pinia store 按域划分，与后端模块边界对齐。
- 不在 store 内做权限判断；权限来自 `/permissions/explain`，store 只缓存结果。
- 不为"未来可能需要"提前堆抽象，按当前真实接口形态写最直接的代码。

## 非目标

- 不维护第二套后台前端。
- 不再回到 mock / 静态元数据驱动模式。
- 不在前端实现权限规则、tenant 概念、API 注册等本应由后端负责的能力。
