## 1. 前端接口级临时缓存体系

### 当前决策

- 当前版本先不实施全站接口级缓存改造。
- 已落地的仅保留：
  - 顶部快捷入口配置缓存
  - 按用户隔离
  - 本地缓存时长 `3 小时`
- 其余接口暂不接缓存，避免在没有统一规则时出现串缓存、脏缓存和失效混乱。

### 目标

- 为前端建立一套统一的临时缓存工具，用于降低非实时接口的重复请求。
- 统一覆盖：
  - 选项/字典类接口
  - 菜单树/页面树/功能包树
  - 运行时导航 manifest
  - 菜单空间配置与当前空间解析
  - 轻量概览、摘要、配置类接口

### 设计原则

- 缓存层必须是前端统一工具，不允许各页面自己随意写 `localStorage`。
- 必须支持：
  - `TTL`
  - `force refresh`
  - `invalidate(key/prefix)`
  - `L1` 内存缓存
  - `L2` 本地缓存
- 缓存键必须支持作用域隔离：
  - 用户
  - 团队/上下文
  - 菜单空间
  - 查询参数

### 建议结构

- 建议目录：
  - `frontend/src/utils/cache/`
- 建议文件：
  - `request-cache.ts`
  - `cache-key.ts`
  - `cache-scope.ts`

### 建议分层

#### L1 内存缓存

- 用途：当前标签页会话内的高频短期复用
- 适合：
  - 用户信息
  - 消息摘要
  - 消息预览
  - 短时间内重复打开的弹窗基础数据

#### L2 本地 TTL 缓存

- 用途：低实时接口的跨页面复用
- 适合：
  - 角色选项
  - 功能包选项
  - 权限动作选项
  - 菜单树
  - 页面选项
  - 菜单空间配置
  - 我的团队列表
  - 运行时导航 manifest
  - 消息派发选项

### 建议缓存时长

| 类型 | 建议时长 | 隔离维度 |
| --- | --- | --- |
| 快捷入口配置 | 3 小时 | 用户 |
| 菜单空间配置 / Host 绑定 | 30 分钟 | 用户 + Host |
| 当前空间解析结果 | 10 分钟 | 用户 + Host + 空间 |
| 我的团队列表 | 10 分钟 | 用户 |
| 运行时导航 manifest | 3 到 5 分钟 | 用户 + 空间 + 团队上下文 |
| 公开运行时页面清单 | 30 分钟 | Host + 空间 |
| 角色选项 | 30 分钟 | 用户 + 上下文 |
| 功能包选项 | 20 分钟 | 用户 + 上下文 + 查询参数 |
| 权限动作选项 | 20 分钟 | 用户 + 上下文 + 查询参数 |
| 菜单树 | 10 分钟 | 用户 + 空间 |
| 页面选项 / 页面菜单选项 | 10 分钟 | 用户 + 空间 |
| API 分类 | 30 分钟 | 用户 |
| API 概览 | 5 分钟 | 用户 |
| 消息派发选项 | 10 分钟 | 用户 + 团队/上下文 |
| 消息摘要 / 预览 | 30 到 60 秒 | 用户 + 团队/上下文 |

### 明确不做缓存的类型

- 主列表数据：
  - 用户列表
  - 角色列表
  - 权限列表
  - API 列表
  - 页面列表
  - 团队列表
- 当前态诊断结果：
  - 访问链路测试
  - 权限诊断
  - 影响预览
  - 删除预览
  - 风险审计
- 强依赖立即一致性的结果型接口：
  - 用户/角色/团队绑定结果
  - 收件箱详情
  - 消息详情
  - 已读/待办状态敏感列表
