# 系统兼容与收紧审计

> 更新时间：2026-03-31。这里只保留仍在运行时有意义的兼容边界。

## 正式保留

- 菜单空间默认回退到 `default`。
- 运行时页面注册保留一次重建与防御性回退。
- 面包屑仍允许从菜单树补父链。

## 已收紧

- 菜单分组、菜单空间候选、消息工作台等失败场景统一走页内提示或空态，不再刷业务式 warning。
- 快捷入口与工作标签的非关键 fallback 只保留开发期日志。
- 前端不再继续扩张“可配置但不生效”的兼容提示。

## 待确认清理

- `frontend-copy` 仍视作历史副本，暂不直接删除。
- 页面中心与菜单主页面的并存解释层先保留，后续只在真实业务进入时再评估继续压缩。
