# 菜单与页面管理当前架构

> 基线日期：2026-03-31。正式实现以 `docs/scaffold-navigation-access-redesign.md` 为总基线，本文只保留稳定边界和落地口径。

## 1. 当前正式结论

当前导航体系采用显式双模型：
- 菜单入口路由：负责导航树、入口路由、外链
- 受管页面：负责详情页、流程页、独立页、共享内页与分组

一句话：
- 菜单管入口，页面管非入口访问单元，权限与空间都统一消费后端编译结果。

## 2. 菜单正式模型

### 2.1 菜单类型

菜单只允许三类：
- `directory`
- `entry`
- `external`

含义：
- `directory`：只做导航层级与分组，不挂组件
- `entry`：叶子菜单自己就是页面入口，直接配置 `path/component/accessMode`
- `external`：只维护展示与 `link`

### 2.2 菜单职责

菜单负责：
- 导航树结构
- 排序、显隐、图标、徽章等展示元数据
- 入口访问模式
- 空间归属与默认首页候选

菜单不再负责：
- 详情页、流程页、配置页的独立登记
- 主页面 / 挂接主页面语义
- 页面空间复制

## 3. 受管页面正式模型

### 3.1 页面类型

页面中心只保留：
- `inner`
- `global`
- `group`
- `display_group`

### 3.2 页面职责

页面负责：
- 菜单入口下的非直达页
- 详情页、流程页、模板页、记录页
- 父子页面链、面包屑链与高亮辅助
- 少量独立页的显式空间暴露

页面不再负责：
- 菜单直达入口页
- 按空间复制同一组件定义

## 4. 菜单与页面边界

### 4.1 挂载关系

受管页面可以：
- 挂到菜单：`parent_menu_id`
- 挂到页面：`parent_page_key`
- 仅做管理分组：`display_group_key`
- 完全独立：无菜单父级、无页面父级

### 4.2 空间归属

空间归属规则按优先级：
1. 菜单父级继承
2. 父页面继承
3. `page_space_bindings` 显式暴露
4. 兼容读取旧 `ui_pages.space_key`
5. 都没有时默认全局共享

## 5. 运行时主链

### 5.1 后端统一入口

正式入口：
- `GET /api/v1/runtime/navigation`

返回内容：
- 当前空间解析结果
- 已裁剪菜单树
- 菜单入口路由
- 受管页面路由
- 上下文与版本戳

### 5.2 前端消费

前端首屏只做：
- 拉用户信息
- 拉 navigation manifest
- 归一化菜单树与受管页面
- 动态注册路由

前端不再重复做菜单/页面权限裁剪。

## 6. 管理页边界

### 6.1 菜单管理页

只负责：
- 菜单树
- 入口类型
- 路由与外链配置
- 管理分组与展示元数据

受管页面关系只允许兼容只读展示。

### 6.2 页面管理页

只负责：
- 非菜单直达页
- 逻辑分组 / 普通分组
- 父子页面链
- 独立页暴露

未注册页面扫描只面向受管页面候选，不再与菜单入口抢同一路由。

### 6.3 菜单空间页

只负责：
- 菜单树视角
- 默认首页
- Host 绑定
- 独立页暴露统计

新增空间初始化只复制菜单树与功能包菜单绑定，不复制页面定义。

## 7. 兼容原则

当前仍保留但降级的内容：
- 旧 `/menus/tree` 与 `/pages/runtime` 兼容接口
- 旧 `ui_pages.space_key` 兼容读取
- 历史菜单直达重复页的迁移回放种子

原则：
- 兼容只读，不再扩张
- 新数据只按新模型写入
