# 菜单与页面管理当前架构

> 基线日期：2026-03-31。本文只保留稳定边界和落地口径。

## 当前正式结论

- 菜单入口路由负责导航树、入口路由和外链。
- 受管页面负责详情页、流程页、独立页、共享内页和分组。
- 菜单管入口，页面管非入口访问单元，权限与空间统一消费后端编译结果。

## 菜单模型

- 菜单只允许 `directory`、`entry`、`external` 三类。
- 菜单负责导航树、展示元数据、入口访问模式、空间归属和默认首页候选。
- 菜单不再负责详情页、流程页、配置页的独立登记。

## 受管页面模型

- 页面类型保留 `inner`、`global`、`group`、`display_group`。
- 页面负责非菜单直达页、父子页面链、面包屑链和少量独立页暴露。
- 页面不再负责菜单直达入口页。

## 边界与空间

- 受管页面可挂菜单、挂页面、做管理分组，或完全独立。
- 空间归属按菜单父级、父页面、`page_space_bindings`、旧 `ui_pages.space_key` 的顺序继承。
- 新数据只按新模型写入，旧兼容只读不扩张。

## 运行时主链

- 正式入口：`GET /api/v1/runtime/navigation`
- 后端返回当前空间解析结果、已裁剪菜单树、菜单入口路由、受管页面路由和上下文版本戳。
- 前端首屏只做拉用户信息、拉 manifest、归一化和动态注册。

## 管理页边界

- 菜单管理页只负责菜单树、入口类型、路由与外链、展示元数据。
- 页面管理页只负责非菜单直达页、分组和父子页面链。
- 菜单空间页只负责菜单树视角、默认首页、Host 绑定和独立页暴露统计。
