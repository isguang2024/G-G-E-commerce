## 2026-03-27 API 固定 GUID 自动注册收口

### 本次改动
- API 自动注册改成按源码固定 GUID 查缺：`apiregistry` 增加了固定 GUID 注册表，走 `apiregistry.Registrar` 的系统 API 在源码里都有明确 `code`，启动时仅按该 GUID 判断是否缺失，不再按 `method+path` 覆盖已有记录。
- 认证、媒体接口也接入了注册器；若后续新增系统 API 但未分配固定 GUID，启动同步会直接跳过该路由，避免再次回退成按路径派生新标识。
- API 管理保存时改为保留原有固定 GUID，避免编辑后重新计算 `code`；“未注册 API”列表也同步改成优先按 GUID 判定，避免数据库里手工改过路径后被误判成新接口。
- API 状态拦截增加对 `/api/v1/api-endpoints` 管理路由的绕过，修复“把 API 停用后连管理页也一起锁死”的问题。
- 已验证：`go test ./...` 通过。

### 下次方向
- 目前固定 GUID 自动覆盖的是走 `apiregistry.Registrar` 的接口；如果后续要把原始 `gin` 直注册路由也纳入自动注册，需要统一接入同一套注册器或显式补固定 GUID。
- 现在自动注册只负责补缺，不负责删除源码里已经移除的旧记录；如果要做清理，建议后续单独补“失效 API 标记/回收”流程，避免误删人工维护数据。
