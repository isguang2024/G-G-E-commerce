## 2. 菜单管理

### 2.1 默认空间过滤

- 账号：平台管理员
- 前置数据：`default` 与 `ops` 两个菜单空间都存在
- 操作步骤：
  1. 登录
  2. 打开 `#/system/menu`
  3. 选择 `default`
  4. 观察列表
- 预期结果：
  - 只显示 `default` 空间菜单
  - 不混入 `ops` 菜单
- 失败现象：
  - 切换空间后仍出现全部空间菜单

### 2.2 菜单分组静默降级

- 账号：平台管理员
- 前置数据：无
- 操作步骤：
  1. 打开 `#/system/menu`
  2. 观察页面在分组接口异常时的表现
- 预期结果：
  - 页面内提示“菜单分组暂不可用，当前按普通菜单树显示”
  - 不弹业务式 warning
  - 分组相关开关与入口会自动禁用或降级
- 失败现象：
  - 一进入页面就弹错误 toast
  - 控制台刷运行期 warning

### 2.3 菜单挂接页面

- 账号：平台管理员
- 前置数据：当前空间下存在可挂接页面
- 操作步骤：
  1. 编辑一个菜单
  2. 选择“挂接主页面”
  3. 保存
  4. 打开 `#/system/page`
- 预期结果：
  - 菜单列表“挂接主页面”列刷新正确
  - 页面管理里能看到同一条挂接关系
- 失败现象：
  - 菜单保存后页面关系不同步
  - 挂接候选串到别的空间

## 5. 消息系统

### 5.1 平台消息发送

- 账号：平台管理员
- 前置数据：发送人、模板、接收组已由演示命令初始化
- 操作步骤：
  1. 打开 `#/system/message`
  2. 确认页面能加载发送配置
  3. 发送一条平台消息
- 预期结果：
  - 请求快速返回
  - 发送记录显示排队中 / 发送中 / 已发送
- 失败现象：
  - 首屏配置加载失败直接弹错
  - 发送后记录页无数据

### 5.2 消息中心收件

- 账号：平台管理员、团队管理员、普通成员
- 前置数据：至少一条平台消息和一条团队消息
- 操作步骤：
  1. 打开 `#/workspace/inbox`
  2. 查看未读数、列表、详情
- 预期结果：
  - 未读数、列表、副标题、详情一致
  - 团队管理员可看到团队消息并标出团队来源
  - 普通成员至少能看到平台演示消息
- 失败现象：
  - 列表有数据但未读数不一致
  - 详情打开失败

### 5.3 富文本内链

- 账号：平台管理员
- 前置数据：存在带 `#/system/page`、`/system/menu` 的已发送消息
- 操作步骤：
  1. 在消息中心打开详情并点击正文链接
  2. 在发送记录详情点击正文链接
  3. 在发送页预览区点击正文链接
- 预期结果：
  - 三处都走统一空间导航策略
  - 单域下跳转正常
- 失败现象：
  - 某一处仍走浏览器默认跳转
  - 某一处忽略空间 / Host 绑定

### 5.4 平台与团队消息入口隔离

- 账号：平台管理员、团队管理员
- 前置数据：`演示团队` 存在
- 操作步骤：
  1. 平台管理员打开 `#/system/message`
  2. 团队管理员打开 `#/team/message`
- 预期结果：
  - 两条入口独立
  - 默认发送对象与可选范围不同
  - 交互结构保持一致
- 失败现象：
  - 团队侧仍看到平台对象
  - 普通成员能进入团队发信页

## 6. 角色场景

### 6.1 平台管理员

- 应能进入：
  - `#/system/menu`
  - `#/system/page`
  - `#/system/menu-space`
  - `#/system/message`
  - `#/workspace/inbox`

### 6.2 团队管理员

- 应能进入：
  - `#/team/message`
  - 团队消息模板/发送记录/发送人/接收组入口
  - `#/workspace/inbox`

### 6.3 普通成员

- 应能验证：
  - 菜单已被裁剪
  - 无法进入系统管理与团队发信入口
  - 无团队上下文，无法进入 `#/team/message`
  - 能在消息中心查看并打开收到的平台消息
