# 脚手架导航 / 页面 / 空间 / 权限体系重构

> 基线日期：2026-03-31。本文是后续实现的唯一设计基线。

## 当前状态

- 菜单入口路由负责导航树、入口路由和外链。
- 受管页面负责详情页、流程页、独立页、共享内页和分组。
- 空间只负责菜单树归属、默认首页、Host 绑定和少量独立页暴露。
- 后端统一通过 `GET /api/v1/runtime/navigation` 编译当前上下文的可访问结果。

## 当前主链

- 菜单类型收口为 `directory / entry / external`。
- 页面类型收口为 `inner / global / group / display_group`。
- 前端首屏只做拉用户信息、拉 manifest、归一化和动态注册。
- 前端不再重复做菜单/页面权限裁剪。

## 当前边界

- 菜单只管入口，不再承担详情页或流程页登记。
- 页面只管非菜单直达页，不再按空间复制同一组件定义。
- 空间初始化只复制菜单树与功能包菜单关联，不复制页面定义。
- 旧接口和旧兼容字段只读，不再扩张。

## 当前落地

- `menus.kind` 已落地。
- `page_space_bindings` 已落地。
- `runtime/navigation` 已接入。
- 前端守卫已切到 manifest 主链。
- 菜单空间初始化已停止复制页面定义。

## 验收矩阵

- 单空间基线下，登录后只靠用户信息和 navigation manifest 就能完成动态路由注册。
- 默认入口无需页面记录也能访问、刷新和高亮。
- 新空间初始化只复制菜单树与功能包菜单关联，不复制页面定义。
- 深链访问、Host 绑定、权限变更和冲突清理都能按统一编译结果收口。
