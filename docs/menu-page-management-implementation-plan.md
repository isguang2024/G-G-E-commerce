# 菜单与页面管理实施现状

> 现状基线：2026-03-27。文件名保留为 implementation-plan，但内容已改为“当前已落地实现 + 剩余边界”，不再保留早期草案写法。

## 1. 已落地后端能力

### 1.1 数据模型

当前已落地的数据模型包括：
- `menus`
- `menu_manage_groups`
- `ui_pages`

页面模型已支持：
- `inner`
- `global`
- `group`
- `display_group`

### 1.2 页面管理接口

当前已落地接口包括：
- `GET /api/v1/pages`
- `GET /api/v1/pages/:id`
- `POST /api/v1/pages`
- `PUT /api/v1/pages/:id`
- `DELETE /api/v1/pages/:id`
- `GET /api/v1/pages/menu-options`
- `GET /api/v1/pages/unregistered`
- `POST /api/v1/pages/sync`
- `GET /api/v1/pages/:id/breadcrumb-preview`
- `GET /api/v1/pages/runtime`
- `GET /api/v1/pages/runtime/public`

### 1.3 运行时缓存

页面运行时链路已补运行时缓存：
- 普通运行时页面注册表缓存
- 公开运行时页面注册表缓存

页面、菜单变更后会主动失效缓存，避免长期读旧结果。

## 2. 已落地前端能力

### 2.1 页面管理页

当前前端已提供正式页面管理页：
- 列表
- 搜索
- 未注册页面同步
- 页面新增/编辑/复制
- 逻辑分组与普通分组管理
- 面包屑预览
- 最终路径预览

### 2.2 路由接入

当前前端路由守卫已接入：
- 菜单动态路由
- 运行时页面路由
- 未登录公开页预注册
- 页面访问模式判断
- 菜单门槛能力判断

运行时页面由 `ManagedPageProcessor` 构造，不再依赖旧菜单“内页类型”。

## 3. 当前正式实现规则

### 3.1 页面是否进入运行时

当前运行时仅注册真正可访问页面：
- `inner`
- `global`

不会进入运行时注册的页面类型：
- `group`
- `display_group`

### 3.2 页面挂载规则

当前实现支持三种挂载方式：
- 不挂载
- 挂到菜单
- 挂到页面

补充规则：
- `display_group` 只做页面管理列表归类
- `group` 参与父子页面链，但自身不生成运行时路由

### 3.3 路径解析规则

当前实现已支持：
- 单段相对路径自动拼接
- 多段绝对路径直接注册
- 外链页面自动落到 `/outside/Iframe`
- 路径预览与实际运行时拼接规则一致

## 4. 与菜单管理的当前关系

菜单侧当前已同步落地：
- `meta.accessMode`
- `menu_manage_groups`
- 菜单编辑页和页面管理页的边界收口

当前正式关系是：
- 菜单页负责入口导航
- 页面页负责访问单元
- 菜单管理分组不影响页面路径
- 页面普通分组不影响菜单权限

## 5. 当前剩余边界

以下内容尚未进一步展开为新专题，但已知仍可继续优化：

- 运行时页面链缺少更细的接口级自动化测试
- 页面与菜单的联动回归仍主要依赖冒烟验证
- 页面运行时当前没有单独的按路径懒匹配接口，仍以注册表为主

## 6. 后续继续改动时的注意点

- 改页面挂载规则时，同时检查：
  - 运行时页面接口
  - `ManagedPageProcessor`
  - 页面表单的最终路径预览
- 改菜单访问模式时，同时检查：
  - 菜单树过滤
  - 公开页预注册
  - 未登录访问链路
- 改页面类型语义时，同时检查：
  - 页面列表分组展示
  - 运行时注册过滤
  - 面包屑和高亮链路

## 7. 当前建议的最小验证

涉及菜单/页面链路改动时，最低建议验证：
- `pnpm exec vue-tsc --noEmit`
- 页面管理页新增/编辑/同步一次
- 公开页未登录访问一次
- 登录后页面挂载、菜单高亮、面包屑预览各走一轮
