# 菜单与页面管理实施现状

> 基线日期：2026-03-31。本文记录当前已经落地的实现与剩余收口项，不再保存旧草案过程。

## 1. 已落地后端能力

### 1.1 数据模型

当前已落地：
- `menus.kind`：显式区分 `directory / entry / external`
- `page_space_bindings`：承载少量独立页空间暴露
- `ui_pages`：仅保留受管页面主表

### 1.2 运行时编译

当前已落地：
- `GET /api/v1/runtime/navigation`
- 页面运行时过滤菜单直达重复页
- 空间解析与页面空间继承统一收口到后端编译链

### 1.3 兼容层

当前仍保留：
- `/api/v1/menus/tree`
- `/api/v1/pages/runtime`
- `/api/v1/pages/runtime/public`

要求：
- 兼容接口结果必须与新 manifest 保持一致
- 不再新增新的双轨语义

## 2. 已落地前端能力

### 2.1 路由守卫

当前已切到：
- 用户信息
- `runtime/navigation`
- 菜单树 + 受管页面动态注册

当前已删除：
- 守卫里对菜单/页面做重复权限裁剪
- 通过路径再做一套前端权限推导

### 2.2 管理页语义

当前已开始收口：
- 菜单管理弹窗按 `directory / entry / external` 编辑
- 页面管理标题明确为“受管页面”
- 未注册页面扫描明确只面向受管页候选
- 菜单空间初始化文案改为“菜单树 + 独立页暴露”口径

## 3. 当前正式规则

### 3.1 菜单入口

- 菜单叶子页优先直接建成 `entry`
- 同路径同组件不再重复建 `ui_pages`
- 外链入口优先用 `external`

### 3.2 受管页面

- 只维护非菜单直达页
- 默认全局定义
- 只有独立页才允许显式空间暴露

### 3.3 空间初始化

- 复制菜单树
- 复制功能包菜单绑定
- 不复制页面定义
- 少量独立页暴露通过 `page_space_bindings` 表示

## 4. 本轮已清理的典型问题

- `team.team -> /team/team` 菜单/页面同路径冲突进入首批标准清理样例
- 菜单种子与页面种子分工重新收口
- 默认首页候选改为直接消费 manifest，而不是前端菜单 + 页面列表自行拼装

## 5. 仍需继续关注的边界

- 菜单管理中的“受管页面关系”仍是兼容只读信息，可继续降级展示层级
- 页面表单中的 `space_key` 仍是兼容输入口，后续可拆成“空间视角 + 独立页暴露绑定”
- 菜单空间页操作仍偏多，后续可以收敛到 `ArtButtonMore`

## 6. 最低验证矩阵

涉及菜单 / 页面 / 空间主链改动时，最低验证：
- `go test ./...`
- `pnpm exec vue-tsc --noEmit`
- 菜单管理、页面管理、菜单空间各走一轮
- 登录后入口菜单刷新、深链访问、默认首页回退各走一轮
