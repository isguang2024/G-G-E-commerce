# 文档导航模板（V5）

用于统一入口文档（`README.md`、`docs/README.md`、`docs/INDEX.md`、子目录 `README.md`）的结构，减少重复和断链。

## 1. 模板目标

- 让读者在 30 秒内找到“先看什么、去哪改、真相源在哪”。
- 入口文档只做导航，不重复实现细节。
- 保证每个链接都可达、每个条目都能落到唯一职责文档。

## 2. 推荐层级

1. 文件定位（它解决什么问题）
2. 快速入口（3~8 个核心链接）
3. 主题索引（按职责分组）
4. 阅读顺序（按角色/场景）
5. 维护约定（更新触发条件与校验动作）

## 3. Markdown 模板

```md
# <标题>

<一句话说明这个入口解决什么问题>

## 快速入口

| 文档 | 作用 |
| --- | --- |
| [A](./A.md) | <职责> |
| [B](./B.md) | <职责> |

## 主题索引

### <主题1>

- [文档1](./x.md)：<一句话>
- [文档2](./y.md)：<一句话>

### <主题2>

- [文档3](./z.md)：<一句话>

## 阅读顺序

### <角色/场景A>

1. <文档路径>
2. <文档路径>
3. <文档路径>

### <角色/场景B>

1. <文档路径>
2. <文档路径>

## 维护约定

- 新增文档时，同步更新本页索引条目。
- 删除文档前，先检索引用并修正链接。
- 若与真相源冲突，以真相源为准。
```

## 4. 适用范围

- 根 `README.md`：仓库总入口。
- `docs/README.md`：文档中枢入口。
- `docs/INDEX.md`：全量索引导航。
- 核心子目录 `README.md`（如 `frontend/`、`docs/guides/`）。
