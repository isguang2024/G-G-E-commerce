# 权限系统总体说明

> 基线日期：2026-03-31。本文只描述当前正式权限架构；导航 / 页面 / 空间的详细方案以 `docs/scaffold-navigation-access-redesign.md` 为准。

## 1. 当前结论

当前权限体系已经收口为：
- 功能包负责业务开通
- 权限键负责动作与 API 原子能力
- 菜单负责导航入口
- 受管页面负责非菜单直达访问单元
- 平台与团队继续通过快照进入运行时读链
- 导航 / 页面访问统一由后端一次编译、前端一次消费

一句话：
- 先开功能包，再由后端编译当前上下文下的菜单、页面与动作结果，前端只消费 manifest。

## 2. 当前正式主数据

### 2.1 用户、角色、团队

- 用户：`users`
- 角色：`roles`
- 团队：`tenants`
- 团队成员：`tenant_members`
- 用户角色关联：`user_roles`

### 2.2 权限与功能包

- 权限键主表：`permission_keys`
- 功能包主表：`feature_packages`
- 组合包关系：`feature_package_bundles`
- 功能包绑定权限键：`feature_package_keys`
- 功能包绑定菜单：`feature_package_menus`

### 2.3 菜单、页面、API

- 菜单：`menus`
- 菜单管理分组：`menu_manage_groups`
- 页面：`ui_pages`
- 独立页空间暴露：`page_space_bindings`
- API 元数据：`api_endpoints`
- API 与权限键绑定：`api_endpoint_permission_bindings`

### 2.4 运行时快照

- 平台用户快照：`platform_user_access_snapshots`
- 平台角色快照：`platform_role_access_snapshots`
- 团队边界快照：`team_access_snapshots`
- 团队角色快照：`team_role_access_snapshots`

## 3. 上下文模型

系统只保留两个正式上下文：
- `platform`
- `team`

规则：
- 无 `X-Tenant-ID` 时走平台上下文
- 有 `X-Tenant-ID` 时走团队上下文
- 团队鉴权前必须确认成员关系与成员状态

## 4. 正式授权模型

### 4.1 功能包

- 基础包直接绑定菜单和权限键
- 组合包只负责组合基础包
- 组合包不直接绑定菜单和权限键

### 4.2 平台侧

平台侧最终能力来自：
- 平台角色功能包并集
- 平台用户直绑功能包
- 平台角色减法：隐藏菜单 / 禁用动作
- 平台用户减法：隐藏菜单

### 4.3 团队侧

团队侧最终能力来自：
- 团队已开通功能包
- 团队边界减法：阻断菜单 / 阻断动作
- 团队角色减法：隐藏菜单 / 禁用动作

### 4.4 本轮不重写的部分

本轮不重写：
- 功能包模型
- 权限键模型
- API 鉴权底座
- 快照刷新机制

本轮只收敛：
- 菜单显隐
- 受管页面访问
- 深链兜底
- 前端重复权限判断

## 5. 导航与页面在权限体系中的位置

- 菜单只控制入口与导航树
- 受管页面只控制非菜单直达访问单元
- 权限键控制动作/API
- 功能包提供候选能力范围

禁止继续混用：
- 菜单 = 页面
- 页面 = 权限
- 前端再推导一套后端未声明的导航权限

## 6. 统一访问编译层

当前正式运行时入口：
- `GET /api/v1/runtime/navigation`

后端会在当前 `user / team / space` 上下文内一次性编译：
- 当前空间解析结果
- 菜单树
- 菜单入口路由
- 受管页面路由
- 动作权限摘要

前端只做：
- 轻量归一化
- 动态路由注册
- 缺失路由时重拉 manifest

## 7. 快照与刷新策略

当前正式策略：
- 运行时优先读快照
- 写链成功后主动刷新受影响主体
- 快照缺失时允许自动重建并持久化
- 导航 manifest 不再绕开快照另算一套前端权限

需要触发刷新的典型变更：
- 功能包绑定关系变化
- 菜单启停、菜单绑定、菜单空间变化
- 角色或用户隐藏菜单 / 禁用动作变化
- 团队开包、团队边界变化
- 团队成员角色变化

## 8. 新模块接入清单

新增模块时至少检查：
1. 是否新增正式 `permission_keys`
2. 是否需要新增基础功能包或纳入组合包
3. 是否需要新增菜单入口，还是只需要新增受管页面
4. 是否需要新增 API 元数据与权限键绑定
5. 是否覆盖相关快照刷新
6. 是否同步更新专题文档

## 9. 当前非目标

- 不恢复用户个人功能权限直配为主链
- 不恢复菜单内页类型为页面管理主模型
- 不让前端自行补一套页面/菜单权限体系

## 10. 当前判断标准

- 看模块是否开通：先看功能包
- 看入口是否展示：看菜单与菜单减法结果
- 看受管页面是否可访问：看后端编译后的页面访问结果
- 看动作/API 是否允许：看权限键与快照
