# 业务模块开发说明

## 定位

`backend/internal/modules` 是业务核心层。系统基础能力集中在 `system/`，新增业务模块应直接复用现有认证、权限、菜单、页面、API 注册和租户上下文能力。

## 新模块最小要求

- 新模块直接放在 `modules/` 下
- 业务表需要按需接入迁移
- 接口注册要补齐 API 元数据
- 平台入口和团队入口需要分开，业务实现共用同一套 service
- 涉及团队数据时，业务表应带 `tenant_id`

## 当前边界

- 菜单权限、功能权限和数据权限不要混用
- 不要让一个接口同时混跑多个作用域语义
- 说明文档优先写到对应模块目录，不要再回堆根目录
