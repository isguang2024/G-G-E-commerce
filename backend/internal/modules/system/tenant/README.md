# 团队与租户说明

## 定位

项目里的“团队”就是租户上下文。平台侧团队管理与当前团队上下文管理并存，但不能混淆。

## 接口边界

- 平台侧接口面向系统管理员或平台角色
- 当前团队接口面向当前团队管理员或团队成员
- 一个接口不要同时混跑多套作用域语义

## 数据职责

- `tenant_members`：成员归属与状态
- `user_roles`：平台角色与团队角色
- `team_access_snapshots`：团队边界运行时主链

## 判断规则

团队作用域下，统一按用户状态、成员状态、团队边界、个人覆盖、团队角色和默认拒绝的顺序判断。

## 新业务约定

- 业务表默认带 `tenant_id`
- 页面入口区分平台入口和团队入口
- 团队内接口用团队作用域功能权限
- 平台管理接口用全局作用域功能权限
