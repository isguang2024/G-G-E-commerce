# System 模块说明

## 定位

`system` 模块是项目基础内核，负责认证、用户、角色、菜单、页面、功能权限、API 注册、作用域与团队上下文。

## 当前划分

- `auth/`：登录、用户信息、当前权限集合
- `user/`：用户管理与用户覆盖
- `role/`：角色、菜单权限、角色功能权限、角色数据权限
- `menu/`：菜单树、管理与运行时返回
- `permission/`：功能权限注册表
- `apiendpoint/`：API 注册表与元数据同步
- `scope/`：作用域定义
- `tenant/`：团队、当前团队、团队边界、团队成员

## 统一判断顺序

1. 用户状态
2. 团队成员状态
3. 当前上下文
4. 团队功能边界
5. 用户个人覆盖
6. 角色权限
7. 默认拒绝

## 边界

- 业务模块尽量复用 `system` 的认证、权限、租户上下文和 API 注册能力
- 不要重新设计一套权限模型
- 数据权限默认不作为新增业务接入的第一优先级
