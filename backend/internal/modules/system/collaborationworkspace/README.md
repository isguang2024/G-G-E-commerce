# collaborationworkspace

这个目录只处理“协作空间实体”和“协作空间成员关系”。

## 当前文件

| 文件 | 说明 |
| --- | --- |
| `service.go` | 协作空间列表、创建、更新、删除、成员增删、成员角色切换 |
| `README.md` | 目录职责和边界说明 |

## 负责什么

- 协作空间基础信息的增删改查
- 协作空间成员增删
- 协作空间成员角色切换
- 协作空间创建后的默认角色绑定和运行时刷新

## 不负责什么

- 不负责统一工作区模型，那个在 `../workspace/`
- 不负责菜单空间和 Host 访问入口，那个在 `../space/`
- 不负责最终权限判定，那个仍走 `permission/` 主链和 evaluator

## 接入边界

- 需要操作成员关系时，优先从这里进入，不要直接在上层 handler 拼仓储逻辑。
- 需要同时处理协作空间实体和工作区映射时，和 `workspace/` 配合，而不是把两套概念混成一个 service。
