# 协作空间模块说明

## 定位

项目里的协作空间是当前业务协作主体。tenant 预留给未来真正的多租户系统，不再作为当前协作上下文命名。

## 接口边界

- 平台侧接口面向系统管理员或平台角色
- 当前协作空间接口面向当前协作空间管理员或协作空间成员
- 一个接口不要同时混跑多套作用域语义

## 数据职责

- `collaboration_workspace_members`：成员归属与状态
- `user_roles`：平台角色与协作空间角色（历史兼容）
- `collaboration_workspace_access_snapshots`：协作空间边界运行时主链

## 判断规则

协作空间作用域下，统一按用户状态、成员状态、协作空间边界、个人覆盖、协作空间角色和默认拒绝的顺序判断。

## 新业务约定

- 业务表默认带 `collaboration_workspace_id`
- 页面入口区分平台入口和协作空间入口
- 协作空间内接口用协作空间作用域功能权限
- 平台管理接口用全局作用域功能权限
