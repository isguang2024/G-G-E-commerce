# 功能权限说明

## 权限分层

- 菜单权限：决定入口是否显示
- 功能权限：决定按钮和接口是否允许执行
- 数据权限：决定数据范围

## 唯一标识

功能权限以 `resource_code` + `action_code` + `scope_id` 作为唯一标识，同一动作在不同作用域下应拆成不同权限定义。

## 作用域

- 只保留 `global` 与 `team`
- 同一功能同时支持平台和协作空间时，权限定义拆成两条，业务实现复用同一套 service

## 权限来源

- `source` 说明权限进入系统的方式
- `feature_kind` 说明权限属于系统功能还是业务功能

## 接入规则

- 菜单继续由角色菜单权限控制
- API 注册时补齐元数据
- 新增功能时先定义权限，再接 API，再接前端按钮
